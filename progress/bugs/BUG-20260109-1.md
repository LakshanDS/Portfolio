# Bug Report: ESLint Errors in Roadmap Page

**Bug ID**: BUG-20260109-1
**Date Reported**: January 9, 2026
**Severity**: Medium (Non-blocking)
**Status**: Open
**Component**: Roadmap Page

## Description

ESLint reports unescaped quotes in JSX string at line 129 of the roadmap page. The quotes in the git commit message are not properly escaped, violating React's `no-unescaped-entities` rule.

## Location

**File**: [src/app/roadmap/page.tsx](d:/Projects/Portfolio/src/app/roadmap/page.tsx#L129)
**Line**: 129
**Column**: 81 and 99

## Error Message

```
D:\Projects\Portfolio\src\app\roadmap\page.tsx
  129:81  error  `"` can be escaped with `&quot;`, `&ldquo;`, `&#34;`, `&rdquo;`  react/no-unescaped-entities
  129:99  error  `"` can be escaped with `&quot;`, `&ldquo;`, `&#34;`, `&rdquo;`  react/no-unescaped-entities
```

## Code Context

```typescript
// Line 129 (Current)
<span className="text-[#137fec] mr-2">$</span>git commit -m "self upgrade init"
```

This line appears in the terminal visual section of the roadmap page, showing a simulated git commit command.

## Expected Behavior

The quotes should be properly escaped using HTML entities to satisfy React's JSX requirements.

## Proposed Fix

```typescript
// Replace with:
<span className="text-[#137fec] mr-2">$</span>git commit -m &quot;self upgrade init&quot;
```

Alternative (using curly quotes):
```typescript
<span className="text-[#137fec] mr-2">$</span>git commit -m &ldquo;self upgrade init&rdquo;
```

## Impact

- **Build Status**: Does NOT block build (build passes successfully)
- **Runtime**: No impact on application functionality
- **Code Quality**: Violates React best practices
- **ESLint Score**: Contributes to failing lint status (2 errors total)

## Reproduction Steps

1. Run ESLint on the project:
   ```bash
   npm run lint
   ```
2. Observe the error output showing unescaped quotes at line 129

## Root Cause

The git commit message contains straight double quotes (`"`) which are treated as JSX syntax rather than literal characters. In JSX, quotes used within JSX expressions should be escaped using HTML entities.
